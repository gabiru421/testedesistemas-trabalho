# Trabalho Avaliativo #1 - Roteiro de Testes

## Descrição

O arquivo apresenta um roteiro de testes para uma tela de Cadastro de Cliente, contemplando cenários positivos e negativos para validação do sistema.

O objetivo é verificar se os dados informados pelo usuário são tratados corretamente e se as funcionalidades da aplicação atendem aos requisitos esperados.

## Testes Realizados

### Cadastro de Cliente

Foram realizados testes para validar:

- Inclusão de clientes
- Validação de CPF
- Validação de nome do cliente
- Validação de e-mail
- Validação de telefone
- Validação de data de cadastro
- Limite de caracteres dos campos
- Campos obrigatórios

### Testes dos Botões

Foram avaliados os seguintes botões:

- Incluir
- Editar
- Excluir
- Confirmar
- Cancelar
- Fechar

### Testes de Limite

Foram realizados testes de limite para:

- Nome
- CPF
- Telefone
- Endereço
- E-mail
- Data

## Cenários Testados

O roteiro contém casos de teste positivos e negativos, verificando:

- Aceitação de dados válidos
- Rejeição de dados inválidos
- Mensagens de erro
- Persistência de dados
- Comportamento esperado das ações do sistema
